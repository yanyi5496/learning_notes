```
用户管理模块
用户注册 在前端给用户展示表单，用户在其中填入自己的相关信息，在用户输入邮箱号之后会通过AJAX
  调用后台接口检查该用户名是否已被注册过，并给出提示。如果没有被注册过那么输入密码等信息后提交
  注册信息到后台，后台通过UUID随机生成一个盐值，然后将用户密码通过盐值进行加密，然后在数据库中
  存储加密后的密码和盐值，系统中不保存用户的明文密码。
  在用户信息存入数据库之后，将调用接口向用户
  的邮箱发送一封包含激活链接的邮件，激活链接由用户邮箱和一串随机字段串组成，该字符串将在调用时
  存储到Redis缓存中。用户访问激活链接，后台根据请求中的用户邮箱从Redis中取出字符串与请求中携带
  的数据进行比对，如果值一直则激活成功，不一致则激活失败。
```

```
用户登录
用户注册了账号且成功激活可以登录到系统中。访问系统会直接进入到登陆页面中，前端通过
  AJAX将用户的用户名和密码发送到后台，后台通过用户名从数据库中取出数据，数据库无数据
  则提示前端账户未注册或未激活，有数据则将用户输入密码和数据库中的盐值做加密运算，然后
  与数据库中的密文密码比对，一致则用户登录成功，否则提示用户密码错误。用户登录成功之后
  后台会设置一个Token作为身份信息存入前端的Cookie中，设置有效期为一小时。
```

```
修改密码
在用户忘记密码需要修改密码的情况下，点击忘记密码链接会跳转到到修改密码界面。用户在表单填入邮箱账户，前端通过AJAX提交邮箱信息
  到后台，检测账号是否存在并激活，给出前端提示。验证通过之后点击获取验证码，此时向前端异步向后台请求，后台根据请求生成一串验证码，
  将验证码发送到用户的邮箱，同时存入Redis，并设置有效时间。用户填入邮箱中收到的验证码和新密码提交所以表单，后台取出Redis中的验证码
  和前端提交的值进行比对，一致则修改成功将用户重定向到登录界面，不一致则修改失败，给出提示。
```
